About the Vehicle Inspection System

This project aims to automate the equipment inspection and maintenance process for the 1100th TASMG MD Army National Guard Unit. The system replaces manual paper-based processes with a streamlined web application, ensuring vehicles are mission-ready efficiently and effectively.

Carousel of Features

Problem Statement

Currently, vehicle inspection and maintenance processes rely on outdated paper forms, causing inefficiencies and delays in fleet dispatch. This web application automates the process, enabling soldiers to input inspection data quickly and update vehicle statuses in real-time.

Key Features

  • Login System: Secure user authentication with role-based permissions using Auth0.
  • Dashboard: Overview of all vehicles with status indicators (e.g., mission-ready, under inspection).
  • Inspection Forms: Dynamic fields based on vehicle type, enabling detailed fault documentation and maintenance tracking.
  • Integration: APIs such as Google Drive for document storage, Google Calendar for scheduling, and IP Stack for location-based access control.

Stakeholders

The primary users are the soldiers, drivers, mechanics, and supply sergeants of the 1100th TASMG MD Army National Guard Unit. This application aims to simplify their workflow and improve fleet readiness.

Technology Stack

  • Frontend: HTML, CSS, JavaScript (with potential use of libraries like React).
  • Backend: JavaScript for API development.
  • Database: To store and manage user roles, vehicle inspection records, and work order data.
  • APIs: Google Drive, Google Calendar, Auth0, and IP Stack.
